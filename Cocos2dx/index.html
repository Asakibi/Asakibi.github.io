Cocos2d-x 4.0 C++ 新手部分問題一覽

　　學校暑期的實訓中，小組決定使用 Cocos2d-x 4.0 進行開發，製作一款「坦克大戰」遊戲。然而，網絡上關於 Cocos2d-x 4.0 的教程寥寥可數，而舊版本的教程常常佔據搜索引擎前幾頁，使得新手開發者難以搜尋相關信息。同時，部分用戶不規範轉載，在内容上疏忽大意，並熱衷於取含糊不清的標題，而平台對此監管不力，對搜索引擎使用者造成了困擾。筆者作爲第一次接觸 Cocos2d-x 4.0 的初學者，在遊戲開發中也遇到了不少問題，經過不斷查找解決思路，有了些許供新手入門的小提示，在此分享出來，以供後繼者參考。須注意的是，筆者只是在短暫的時間内接觸了 Cocos2d-x 4.0，既無及時的記錄和反饋，也無深入的理解和思考，僅僅是不求甚解而已。
　　筆者的開發環境：Cocos2d-x 4.0 （C++），Tiled Map Editor 1.4.1，VS 2019，Windows 10。後來用到了 SQLite3。

繼承自 Sprite 類的問題

  1. 用子類繼承 Sprite 類。
  2. 子類中加入 create() 函數。 其格式可以照搬上面所寫出的 create() 函數格式。
  3. 子類中加入 init() 函數（或者 initWithFile() 函數或 initWithTexture() 函數（統稱爲初始化函數），這取決於你想用哪種方式創建該子類精靈對象）。如果是採用 init() 函數作爲初始化函數，那麽第二步的 create() 函數可以寫成之前所説的宏定義。如果不採用 init() 函數，應該把之前的 create() 函數中的 init() 替換爲你需要的函數，並添加對應的參數使得編譯能夠通過。

Tiled Map Editor 相關問題

關於某文件 479 行 compression 斷言

　　導入地圖文件後，發現彈出窗口報錯，指出某文件 479 行 compression 斷言錯誤：

　　這是由於在 Tiled Map Editor 創建瓦片地圖時，沒有更改瓦片地圖的壓縮方式。Cocos2d-x 4.0 的瓦片地圖只支持無壓縮、gzip 壓縮和 zlib 壓縮，而 Tiled Map Editor 的默認壓縮方式是「Z標準壓縮」。
　　正確的修改方法，是在 Tiled Map Editor 中選擇地圖 -> 地圖屬性，將「圖塊層格式」設定爲「Base64 無壓縮」「Base64 gzip壓縮」或「Base64 zlib壓縮」。

關於地圖無法在遊戲界面正常顯示的問題

　　筆者在製作地圖時，先創建了 .tsx 文件，然後以此作爲圖集來創作地圖。但在導入地圖，並將地圖添加到某個場景的節點之後，地圖在遊戲界面上的顯示非常奇怪。由於無現成的錯誤地圖，只能憑印象描述一下：地圖只顯示有規律的色塊，看得出輪廓但材質極度損害。
　　但若直接向 Tiled Map Editor 的「圖塊集」窗口中拖動圖片文件，選擇類型爲「基於整張圖塊集圖片」，並勾選「嵌入地圖」，這樣產生的瓦片是可以在 Cocos2d-x 中正常顯示的。
